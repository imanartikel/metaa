Privacy Policy

Last updated: May 14, 2026

Meta Ad Drafter is a private/local tool used to help draft and manage Meta advertising assets through the Meta Marketing API. This tool is designed for internal use and does not sell personal information.

Information We Access

When connected to Meta, the tool may access information required to operate advertising workflows, including Meta ad account identifiers, Page identifiers, Page names, ad account names, uploaded image creative metadata, and API response metadata.

How We Use Information

Information is used only to:

  • validate Meta API credentials;
  • read accessible ad accounts and Pages;
  • upload image creatives;
  • create draft ad creative objects for manual review;
  • store local logs for debugging and audit purposes.

Advertising Safety

The tool does not automatically publish active ads or spend advertising budget. Any future campaign, ad set, or ad creation workflow is intended to default to PAUSED and require manual review before publishing.

Data Storage

API responses and draft artifacts may be stored locally on the device running the tool. Access tokens and app secrets are stored in local environment variables and are redacted from generated logs.

Data Sharing

We do not sell, rent, or share data with third parties for their own marketing purposes. Data may be sent to Meta only as necessary to use the Meta Marketing API. If AI generation providers are added in the future, only the minimum creative brief information necessary to generate draft copy or image prompts should be sent to those providers.

Data Deletion

Users may delete locally stored logs, draft files, and generated assets from the local project folder at any time. To revoke Meta access, remove the access token from the local environment file and revoke the app or token from Meta account settings.

Security

Reasonable local security practices are used, including environment variables for credentials and redaction of sensitive values in logs. Users are responsible for keeping their local machine, access tokens, and cloud credentials secure.
